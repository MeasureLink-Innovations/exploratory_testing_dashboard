## Design Context

### Users
Testers navigating a dual-mode workflow: reflective and structured during session preparation/debriefing, but high-pressure and fast-paced during test execution. The interface must transition seamlessly between providing deep focus and enabling rapid data capture.

### Brand Personality
**Light, Precise, Reliable.** The tool should feel like a high-end instrument—essential, accurate, and completely transparent in its operation.

### Aesthetic Direction
**E-Ink / Paper Style.** High contrast, intentional use of whitespace, and a focus on legibility. Think architectural drawings or premium e-readers.
- **Visual Direction**: Brutalist-adjacent but refined. Heavy focus on grid alignment and utilitarian table layouts.
- **Anti-References**: Avoid the "cockpit" problem (too many metrics, buttons, and status indicators visible at once). 

### Design Principles
1. **Zen Execution**: When in-progress, minimize all non-essential UI to reduce cognitive load during high-pressure testing.
2. **Alignment as Information**: Use strict mathematical alignment and consistent spacing to communicate precision and reliability.
3. **Paper-Thin Layers**: Avoid complex shadows or depth. Use borders, subtle fills, and line weights to create hierarchy, staying true to the E-Ink aesthetic.
4. **Persistent Simplicity**: Every state change (like theme or version selection) should be handled via minimal, low-friction interactions (e.g., a small slider instead of a complex toggle).
